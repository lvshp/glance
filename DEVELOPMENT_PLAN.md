# 待开发计划

这个文档记录后续体验优化方向。优先做用户能直接感知、风险较低的改进；较大的性能和架构项放在后面分阶段处理。

## P0：近期优先

- [ ] 优化精简阅读底栏
  - 目标：让精简模式下的章节和进度更直观。
  - 建议展示：`第一百二十八章  7%  120/1680`。
  - 验收：精简阅读页底部能同时看到章节名、百分比、当前位置/总量。

- [ ] 首页显示阅读界面模式状态
  - 目标：避免在书架页误以为精简模式没有生效。
  - 建议：在首页右侧详情或 footer 中显示 `阅读界面：精简 / 全信息`。
  - 验收：首页不切换布局，但能明确看到阅读页当前会使用哪种界面模式。

- [ ] 书架搜索
  - 目标：书籍变多后能快速定位书名。
  - 建议：在书架页按 `/` 输入关键字，按 `Esc` 取消搜索。
  - 验收：搜索结果只过滤书名匹配项，并保留当前排序逻辑。

## P1：阅读体验

- [ ] 阅读页快捷浮层
  - 目标：在阅读页快速查看常用键位，尤其照顾精简模式。
  - 建议：按 `?` 在底部或居中显示小型快捷键浮层，再按一次隐藏。
  - 验收：不跳出阅读页，不遮挡过多正文。

- [ ] 自动隐藏临时状态消息
  - 目标：避免 `已切换...`、`已保存...` 这类状态一直占据状态栏。
  - 建议：临时状态 2-3 秒后恢复为空或默认状态。
  - 验收：长期状态和错误状态不被过早清掉。

- [ ] 更灵活的跳转输入
  - 目标：阅读页可快速跳到章节或百分比位置。
  - 建议：支持类似 `g 50%`、`g 128` 的输入。
  - 验收：能跳转到百分比位置或章节编号，输入错误有明确提示。

## P2：启动与命令行体验

- [ ] 启动参数支持精简阅读
  - 目标：命令行直接打开书时可以直接进入精简阅读。
  - 建议：增加 `--compact` 参数。
  - 验收：`readcli --compact book.epub` 打开后直接使用精简阅读界面。

- [ ] 继续阅读入口强化
  - 目标：首页更突出最近阅读和继续阅读信息。
  - 建议：右侧详情强化当前章节、上次时间、进度条。
  - 验收：首页选中书籍后能一眼看出从哪里继续。

## P3：性能与长期优化

- [ ] EPUB 解析缓存落盘
  - 目标：大 EPUB 第二次打开更快。
  - 建议：把解析后的章节文本和目录缓存到数据目录，用文件 size/mtime 校验失效。
  - 验收：同一本未修改 EPUB 二次打开明显变快；文件变化后能自动重建缓存。

- [ ] 保存失败提示集中化
  - 目标：让配置、书架、进度、书签保存失败的提示更统一。
  - 建议：保留当前状态栏提示，同时考虑集中记录最后一次持久化错误。
  - 验收：失败时可见、成功时安静，不干扰正常阅读。
